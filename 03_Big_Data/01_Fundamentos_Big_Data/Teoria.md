# Contenido Teórico: Fundamentos de Big Data
## Big Data - Bloque 1

## 1. Introducción a Big Data

### 1.1 Definición
Conjunto de datos cuyo tamaño, velocidad y complejidad requieren tecnologías y estrategias especializadas para captura, almacenamiento, análisis e interpretación.

### 1.2 Historia
- **2000s**: Término "Big Data" acuñado
- **2008-2010**: Hadoop se populariza
- **2010s**: Cloud computing y Spark emergen
- **2020+**: Edge computing e IA integrada

### 1.3 Diferencia con Procesamiento Tradicional
- Volumen: De GB a TB/PB
- Velocidad: Procesamiento casi real-time
- Complejidad: Múltiples formatos y fuentes
- Infraestructura: Distribuida, escalable

## 2. Las 5 Vs del Big Data

### 2.1 Volumen
- Cantidades masivas de datos
- Medido en terabytes, petabytes
- Requiere almacenamiento distribuido

### 2.2 Velocidad
- Generación y procesamiento rápido
- Batch vs Stream processing
- Toma de decisiones en tiempo real

### 2.3 Variedad
- Múltiples formatos y fuentes
- Datos estructurados y no estructurados
- Desafío: integración y limpieza

### 2.4 Veracidad
- Calidad y confiabilidad de datos
- Consistencia entre fuentes
- Validación y limpieza crítica

### 2.5 Valor
- Insights y conocimiento actionable
- ROI del análisis
- Decisiones basadas en datos

## 3. Arquitectura de Big Data

### 3.1 Capas Tecnológicas
- **Recolección**: IoT, APIs, logs
- **Almacenamiento**: HDFS, Data Lakes
- **Procesamiento**: MapReduce, Spark
- **Análisis**: SQL, ML frameworks
- **Visualización**: Dashboards, reportes

### 3.2 Modelos de Arquitectura
- Lambda (batch + real-time)
- Kappa (stream processing)
- Modern Data Stack (cloud-native)

## 4. Tecnologías Principales

### 4.1 Hadoop
- Framework para procesamiento distribuido
- HDFS (almacenamiento)
- MapReduce (procesamiento)
- Batch processing

### 4.2 Apache Spark
- Engine de procesamiento más rápido
- Soporte batch y streaming
- APIs en Scala, Python, Java, R

### 4.3 Bases de Datos NoSQL
- MongoDB: documentos flexibles
- Cassandra: alta disponibilidad
- Redis: caché en memoria

### 4.4 Cloud Platforms
- AWS: EC2, S3, EMR
- Azure: DataLake, Synapse
- Google Cloud: BigQuery, Dataflow

## 5. Casos de Uso
- Análisis de redes sociales
- Recomendaciones personalizadas
- Detección de fraude
- IoT y análisis de sensores
- Análisis financiero en tiempo real
